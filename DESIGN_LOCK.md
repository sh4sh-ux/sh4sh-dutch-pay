# Dutch Pay UI Lock

이 문서는 기능 수정 중 화면이 다시 틀어지는 일을 막기 위한 고정 기준입니다.
수정할 때는 아래 기준을 먼저 확인하고, 모바일 화면에서 깨지면 완료로 보지 않습니다.

## 공통 기준
- 기능 로직은 유지하고, UI/레이아웃만 한 화면씩 안정화한다.
- 기준 모바일은 iPhone 14 Pro Max 세로 화면이다.
- 금액은 항상 숫자와 단위를 분리한다. 숫자는 선명하게, `원`은 숫자보다 작고 얇고 옅게 표시한다.
- 금액, 사람 이름, 버튼 텍스트는 세로로 쪼개지면 실패다.
- 같은 의미의 금액은 화면이 달라도 같은 크기, 굵기, 색상 체계를 쓴다.
- 공제는 박스나 알약 형태로 강조하지 않는다. `공제`는 회색, 숫자만 빨간색, `원`은 작고 옅게 표시한다.

## 리포트 화면
- 상단 `정산 내역` 큰 박스는 표시하지 않는다.
- 총 지출 박스는 `총 지출`, 금액, 공제를 한 덩어리로 보여준다.
- 총 지출 금액과 공제는 한 줄에서 읽히게 한다. 너무 좁으면 공제만 자연스럽게 아래로 내려가도 되지만 숫자가 흐려지면 실패다.
- 참여자와 1인 평균만 요약 박스로 보여준다. 송금 건수 요약 박스는 숨긴다.
- 송금 섹션 제목은 아래 리스트의 번호 원형 왼쪽 기준과 맞춘다.
- 송금 금액 오른쪽 끝은 리스트 금액의 오른쪽 기준과 맞춘다.
- 사람별 정산의 `부담`, `결제`, 공제, 결과 금액은 서로 겹치거나 세로로 쪼개지지 않는다.

## 지출 추가
- 사용자가 지금 입력할 대상이 금액임을 바로 알아야 한다.
- 현재 입력 차수는 명확히 표시한다. 예: `3차 금액 입력`.
- 이미 저장된 차수와 이번 입력 차수가 헷갈리지 않게 보조 문구를 둔다.
- 장소는 넓게, 결제자는 이름보다 조금 넓은 안정적인 폭으로 둔다.
- `계속 입력`과 `저장` 버튼은 같은 크기로 둔다.

## 모임 준비
- 모임명은 기본 흐름에서 받지 않는다.
- 기본은 단일 일자이고, 여행처럼 며칠짜리 정산만 기간 정산을 켠다.
- 참여자는 쉼표, 슬래시, 줄바꿈으로 여러 명을 입력한 뒤 `추가`를 눌러 한 번에 등록한다.
- 쉼표나 슬래시를 누르는 순간 바로 등록하지 않는다.

## 아이콘
- 현재 단순한 라인 아이콘 방향은 유지한다.
- 전체적으로 선은 얇게 유지한다. 대략 24px 뷰박스 기준 1.8-1.95px 정도를 우선 기준으로 본다.
